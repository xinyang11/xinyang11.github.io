JAVA运行机制

JAVA的三大基本特性

  • 封装

    将客观事物封装成一个类，也可以是将类中的成员变量或方法进行多种级别的保护，可实现信息的隐藏，防止在对象操作过程中的这些变量造成错误的修改。

  • 继承

    子类对父类的复写，子类可以使用父类的变量或方法而无需再次编码。

  • 多态

    需符合三个条件：1.类继承或者接口实现。2.子类对父类方法进行重写。3.父类接口引用子类对象。

    同一种方法对不同的对象有不同的解释，会产生不同的执行结果。

Java的无平台性

  • Class文件

    Class文件在每个平台都可以使用，且它使用的是字节码编码，与JVM一致，可以统一被JVM处理转化为二进制文件。

  • Java的语言规范

    Java数据类型的值域和方法都是固定的，不会应为平台的变化而变化。对应的C++，它的数据类型与占位宽度有关，同样的int类型数据在32位中为2个字节，在64位中可能就变成了4个字节。

  • Java虚拟机（JVM）

    Java的无平台性很大一部分依靠的是JVM，JVM本身是跟平台相关的。因为各个平台的机器指令可能不同，对应的JVM也就不同。所以针对不同的平台会有对应的JVM，它能将与自己存储单元一致，即都为字节码的文件转化为在当前平台可直接使用的二进制文件。它其实就是起到了平台与class文件间的桥梁的作用。

    这里也能看出来其实JVM不止支持Java语言，同时他还支持所有能生成字节码文件的语言。

浮点型

  • 定义：在计算机科学中，浮点是一种对于实数的近似值数值表现法，由一个有效数字（即尾数）加上幂数来表示，通常是乘以某个基数的整数次指数得到。以这种表示法表示的数值，称为浮点数（floating-point number）。

  • 为什么在涉及金额问题时我们不能考虑使用浮点型

    因为根据定义我们知道计算机中的浮点型是一种对于实数的近似值数值，按照现实情况我们不应该用一个近似值来表示金额。所有我们只能用Long或BigDecimal来表示金额。

字符串

  • String

    一旦一个String对象在内存堆中被创建，他就无法被修改。String类的所有方法都无法改变它本身的值，而是会创建一个新的内存空间，并返回一个新的对象。

  • StringBuffer/StringBuilder

    因为Java特有的垃圾回收机制导致每次都需要清理修改String对象后的原对象。这样会对时间造成极大地浪费。StringBuffer/StringBuider是为了解决String本身无法被修改的情况而产生的。

    • StringBuffer和StringBuilder的区别

      StringBuffer的所有属性、方法都用synchronized修饰，因此它是线程安全的，所以他比较适合用在多线程中，同时它的toString方法会通过使用缓存区的值来构造字符串。相反StringBuilder并不是线程安全的，他适合在单线程中使用，且它的toString方法是通过直接复制成员属性的value值来构造字符串。

      当然StringBuilder的性能肯定是优于StringBuffer的。

  • 方法：

    • substring

      • JDK 6

        substring方法在JDK6中存在内存泄露问题，原因是该方法并没有对字符串进行修改，仅是对string对象内的offset和count属性做了修改，这样做会导致我们的string在使用过程中一直引用的并不是我们实际想要截取的字符串，且由于一直被引用导致无法被回收，这就很有可能造成内存泄漏。

      • JDK 7

        在原来基础上更改了内置属性value[]的值，引用的将是我们实际所截取到字符串，避免了内存泄露。

    • replace、replaceFirst、replaceAll

      这三个的区别在于replace中的target不使用正则表达式，因此仅是普通的字符匹配，而后面两个的target使用正则表达式，replaceFirst 是取代第一个匹配成功的子字符串，而replaceAll取代所有匹配成功的子字符串。

    • String对“+”的重载

      String s=”a”+”b”,编译器进行常量叠加，变成“ab”

      编译期常量之间的运算会进行常量叠加

      编译期常量：

      - 数字或者字符串字面量。
      • 编译期常量赋值的Final修饰的常量
        • 编译期常量简单运算的结果的字面量。

      优化方法：

      可用StringBuilder的append()方法取代，再用toString()方法。

    • 字符串拼接的几种方式和区别

      • 使用“+”拼接字符串

        Java不支持运算符重载，这里仅提供了一个语法糖。

      • concat

        String.concat(str,str);

      • StringBuffer/StringBuilder 的append()方法

        StringBuffer.append(str);

      • 开源类库中的拼接方法

        StringUtils.join

      以上几种连接方式我们往往使用StringBuilder。

      不使用“+”是因为每次使用“+”进行连接时他都会将两边字符串转化为StringBuilder.append进行连接,这样的效率很低。

​ 耗时排名：StringBuilder<StringBuffer<concat<+<StringUtils.join

  • String.valueOf和Integer.toString的区别

    无任何区别，实际上String.valueO就是调用的Integer.toString方法

  • switch对String的支持

    Java中swith支持byte、short、int、char、String

    但实际上swith是只支持int类型的，之所以可以使用char、String是因为它把它们都作为int处理。

    char可以转化为ASCII码。

    String可以转化为hashCode哈希值，然后在通过equals方法进行安全检查。

  • 字符串池、常量池（运行时常量池、Class常量池）、intern

    • 全局字符串池：内部为表结构，使用的是hashCode作为StringTable，存储字符串的直接引用值，每个JVM都仅有一个。
    • Class常量池： Class常量池存在于每一个Class文件中，Class文件开始装载时，Class常量池中的常量使用符号引用代表实际地址。运行时会将符号引用转存到运行时常量池中。Class解析完毕后，将符号引用转化为直接引用，与全局字符串池的StringTable中的引用值对应。
    • 运行时常量池：jvm就会将class常量池中的内容存放到运行时常量池中，由此可知，运行时常量池也是每个类都有一个。
    • intern: intern()函数，返回StringTable中对应字符串的引用值，如果没有就将对应字符串的引用值添加进去，

    ​ str1对象实例化，实例化对象存储在堆中，同时将“abc”字符串的引用值存储在StringTable中。

    ​ str2对象实例化，此时它通过new在堆中开辟了一个新的空间存放str2对象，同时将“def”字符串引用存到StringTable中。

    ​ str3同str1。

    ​ str4被str2.intern()赋值，str2.intern()表示获取str2字符串在StringTable中的直接引用值。因此str4=“def”。

    ​ str5同str1。

基本数据类型

  • 种类：char、byte、short、int、long、float、double、boolean 共有八种。（这些类型都不是面向对象的）

  • 为什么我们需要用设置基本数据类型？

    Java是面向对象的，同时对象都存储在堆中，我们需要通过栈的引用来使用这些对象。因此对象本身十分消耗资源。

    这种时候若我们对一些经常使用的变量仍需要建立对象时，就会显得十分笨重。所以Java就设置了基本数据类型，我们不需要new一个基本数据类型的对象，他们也就不会在堆上创建而是直接在栈中保存，这样就显得十分高效。

包装类型

  • 为什么需要包装类

    因为java是面向对象的，很多时候我们需要用到的是对象而非基本数据类型。因此我们就在每个基本数据类型上都建了一个包装类型，他们具有对象的性质，并添加有属性和方法。

基本数据类型 包装类
byte Byte
boolean Boolean
short Short
char Character
int Integer
long Long
float Float
double Double

  • 基本数据类型与包装类之间的转化

    实际上的转化过程并不需要我们直接编写，因为Java会帮我们自动拆装箱。具体体现在：

    • 基本数据类型放入集合类
    • 包装类和基本类型比大小
    • 四则运算
    • 等等。。

    在拆箱过程中应注意包装类的值，因为包装类可以为null，此时他的拆箱结果会报异常。

  • 自动装箱的缓存机制

    以Integer为例：Integer的缓存存储范围在-128-127之间，也就是说当我们连续新建两个Integer对象时若他们值的范围都在-128-127之间，那么缓存区会存储第一个Integer对象，而第二个Integer对象不会在堆中新建一个存储空间而是使用缓存区的Integer对象。

    这么做的好处当然是提高性能了。

    同时应注意的是包装对象的大小比较最好还是使用equals函数，==仅在范围都是-128-127的时候在有效。

###集合类

  • ArrayList和LinkedList和Vector的区别

    都实现List接口，主要区别在于实现方式以及不同操作下的不同效率

    ArrayList是可变长数组，初次add时它的容量为10，每次扩容都是原来容量的1.5倍。LinkedList是一个双链表，再添加删除方面有较好的的性能。Vector与ArrayList类似，但它属于强同步类，是线程安全的，每次扩容增长原来的一倍。若程序本身线程安全则使用ArrayList即可。

    这里可以给ArrayList分配较大初始值，这样可以有效减少调整大小的开销。

  • SynchronizedList和Vector的区别

    1. SynchronizeList主要使用的是同步代码块包上的list方法，Vector主要使用同步方法。

    2. SynchronizedList有很好的扩展和兼容功能。他可以将所有的List的子类转成线程安全的类。

    3. 使用SynchronizedList的时候，进行遍历时要手动进行同步处理。

    4. SynchronizedList可以指定锁定的对象。

  • HashMap、HashTable、ConcurrentHashMap区别

    • HashTable继承dictionnary类，它的所有方法都用synchronized修饰，因此它是线程安全的。它不可以存储NULL值。内部也是通过单链表解决冲突问题。它的默认大小是11，增加方式为2*old+1。

    • HashMap使用了Map接口，它不是线程安全的，插入操作时可能会出现数据覆盖。但他可以存储NULL键值对。

      • JDK版本1.7时，它的底层数据结构是数组和单链表entry。在put是会有一个resize的过程，这个过程有可能造成头插，这可能造成死循环。1.8后用的尾插。

      • JDK版本1.8时，底层数据结构是数组加单链表或者红黑树。当单链表的长度大于等于8且hash桶大于等于64时他会将单链表转化为红黑树。红黑树节点数量若小于等于6则会转化为单链表。hash桶数量默认为16，加载因子默认为0.75。当哈希表中元素数超出加载因子与当前容量（hash桶数量）乘积时对该哈希表进行resize操作（扩容）。

      • 哈希冲突

        若干Key值进行与运算在对length取模后的值若落在同一个数组下标上，这时出现哈希冲突，需要在链表上使用equals进行进一步判断。

    • ConcurrentHashMap

      ​ jdk1.8之前：ConcurrentHashMap对桶数组进行了一个分段，并对每一个分段都用锁进行保护。所以他是线程安全的。

      ​ 1.8：和HashMap一样的数据结构：数组加单链表或红黑树。渐渐放弃使用分片锁机制而使用Synchronize和CAS进行操作。因为JVM对Synchronize优化较好。

      • CAS：比较并替换，是一种实现并发算法的常见技术。CAS 的使用流程通常如下：1）首先从地址 V 读取值 A；2）根据 A 计算目标值 B；3）通过 CAS 以原子的方式将地址 V 中的值从 A 修改为 B。

        CAS指令执行时，当且仅当内存地址V的值与预期值A相等时，将内存地址V的值修改为B，否则就什么都不做。整个比较并替换的操作是一个原子操作。

        并发量比较大时，它会陷入一个忙循环过程，对CPU消耗较大。建议直接使用锁。

        同时他可能会产生ABA问题，即他在读取值A以及通过原子方式将V中的A值改为B值这两个时期之间就发生了A->B->A的过程，此时CAS会误认为A值从来没有被改变过。（之前读和过段时间读的中间被人修改过后又更改回来了）可以通过加时间戳或者版本号的方式避免这个问题 。

  • Set如何保证元素是不重复的

    • TreeSet

      二叉树实现。数据自动排序，不允许放入null值。底层由TreeMap和keySet实现。其中TreeMap基于红黑树实现，它按key排序。通过compareTo()来判断重复元素。

    • HashSet

      哈希表实现。数据无序，允许放入null值。基本操作由HashMap底层实现，add方法加入数据时，key值覆盖value值保持原样（固定值）。

  • Java8中stream的相关用法

    stream 提供一种对 Java 集合运算和表达的高阶抽象。

    它具有如下特性：

    - 无存储：他不是一种数据结构，仅是一个视图。
- 函数式编程：对数据源的Stream操作并不会更改数据源的值。
- 惰式编程：只有到用户需要最终结果时，流上的操作才会被执行。
- 可消费性：最终操作完后便不可再用，需要再次生成。

    stream分三个阶段：

    • 创建阶段：集合的steam()方法或者直接通过steam类

  • 中间阶段：

    • filter：过滤操作

    • map：映射操作

    • limit\skip：limit是仅取stream中的前n个元素；skip是去掉前n个元素。

    • sorted：排序

    • distinct：去重

  • 最终阶段：

    • forEach：依次遍历
    • count：计数
    • collect：归纳汇总

  • Collection和Collections的区别

    Collection 是一个集合接口，它提供了许多对集合对象操作的通用接口方法。是list、set的父接口。

    Collections 是一个包装类，它不能被实例化。但内部包含许多集合操作的静态方法。

  • Arrays.asList获得的List使用时需要注意什么

    Arrays工具类的asList方法获得的并不是util下的List而是Arrays内部的类。它不具备util下List的一些方法。

    使用时需要通过构造器将其转变为ArrayList

  • Enumeration和Iterator区别

    Enumeration：枚举，只能读取不能修改。只能遍历Vector、HashTable这些集合。

    Iterator：迭代器，即能读取也能修改。支持fail-fast机制。

  • fail-fast和fail-safe

    快速失效系统fail-fast：在Java中即为错误监测机制。

    安全失效fail-safe：先复制原来的内容，在复制的内容上进行遍历。可在多线程并发下使用。
