5548. 最小体力消耗路径

你准备参加一场远足活动。给你一个二维 rows x columns 的地图 heights ，其中 heights[row][col] 表示格子 (row, col) 的高度。一开始你在最左上角的格子 (0, 0) ，且你希望去最右下角的格子 (rows-1, columns-1) （注意下标从 0 开始编号）。你每次可以往 上，下，左，右 四个方向之一移动，你想要找到耗费 体力 最小的一条路径。

一条路径耗费的 体力值 是路径上相邻格子之间 高度差绝对值 的 最大值 决定的。

请你返回从左上角走到右下角的最小 体力消耗值 。

提示：

  • rows == heights.length
  • columns == heights[i].length
  • 1 <= rows, columns <= 100
  • 1 <= heights[i][j] <= 10^6

  • 思路

    矩阵长宽最大值为100，不适合使用dfs遍历(一般在<=20时适宜)。所以使用BFS算法遍历整个矩阵。

    • BFS+DP：不同路径下有不同的最优解，这里需要一个数组dp[i][j]来维护和更新这些最优解。

    • 并查集: 可以将每个矩阵上的方格看做一个点，取出所有的边，并对这些边进行从小到大的排序。并查集依次遍历直到所有点都在同一个集合中。

    • 二分：最大值为10^6，我们可以知道最终结果必然在[0,10^6]这个范围之间。利用二分法依次判断绝对值之差小于等于mid的值的所有点是否能作为起点到终点的一条完整路径。
