    Linux

    一、简介

    Linux 中一切皆文件

    1. 使用原因

    1. 免费
    2. 很多软件原生是在Linux下运行的，庞大的社区支持，生态环境好。
    3. 开源，可被定制，开放，多用户的网络操作系统。
    4. 相对安全稳定

    2. 目录

    目录呈树状结构，根目录为/

    系统所创建的各个目录都有其特殊的意义：

    • bin 存放二进制可执行文件(ls,cat,mkdir等)
    • boot 存放用于系统引导时使用的各种文件
    • dev 用于存放设备文件
    • etc 存放系统配置文件
    • home 存放所有用户文件的根目录
    • lib 存放跟文件系统中的程序运行所需要的共享库及内核模块
    • mnt 系统管理员安装临时文件系统的安装点
    • opt 额外安装的可选应用程序包所放置的位置
    • proc 虚拟文件系统，存放当前内存的映射
    • root 超级用户目录
    • sbin 存放二进制可执行文件，只有root才能访问
    • tmp 用于存放各种临时文件
    • usr 用于存放系统应用程序，比较重要的目录/usr/local 本地管理员软件安装目录
    • var 用于存放运行时需要改变数据的文件

    3.shell 变量

    Shell 变量大致可以分为三类：

    • 内部变量：由系统提供，用户只能使用不能修改。

      • ?
      • GROUPS

    • 环境变量：这些变量决定了用户工作的环境，它们不需要用户去定义，可以直接在 shell 中使用，其中某些变量用户可以修改。

    • 用户变量：由用户建立和修改，在 shell 脚本编写中会经常用到。

      • 变量赋值（定义变量）

        • varName=Value
        • export varName=Value

      • 引用变量$varName

    Shell变量的作用域：

    • 局部变量的作用范围仅仅限制在其命令行所在的Shell或Shell脚本文件中；
    • 全局变量的作用范围则包括本Shell进程及其所有子进程。
    • 局部变量与全局变量互换：可以使用 export 内置命令将局部变量设置为全局变量。 可以使用 export 内置命令将全局变量设置为局部变量。

    权限管理

    root 用户对文件的读写不受权限影响，理论上可以对任何文件进行读写

    文件权限检查时先判断用户名再判断用户所属组，再判断为其他组。

    文件的类型：

    • d： 代表目录
    • -： 代表文件
    • l： 代表软链接（可以认为是 window 中的快捷方式）

    Linux 中权限分为以下几种：

    • r：代表权限是可读，r 也可以用数字 4 表示
    • w：代表权限是可写，w 也可以用数字 2 表示
    • x：代表权限是可执行，x 也可以用数字 1 表示

    修改文件权限指令：chmod

    u：所有者，g：所有组，o：其他人，a：所有人

    chmod [option] u=rwx,g=rwx,o=rwx 文件名

    chmod [option] 777 文件名

    常用选项：

    • -R ：递归修改目录及目录下所有文件权限

    进程管理

    进程基本介绍
    1. 每个执行的程序都称为一个进程，每个进程都被分配有一个IP号。
    2. 每个进程都对应有一个父进程
    3. 每个进程都可能以两种方式存在，即前台和后台。
    4. 一般系统服务都是以后台进程方式存在，且都会驻存在系统中，直到关机才结束。
    显示系统执行进程的方法

    ps : 用来查看当前系统中，有哪些进程正在执行，以及他们的执行情况。

    • -a ：显示当前终端的所有进程信息
    • -u ：以用户格式显示进程信息
    • -x ：显示后台进程运行的参数
    • -f ：以全格式显示(包括显示父进程ppid)
    • -e ：显示所有进程

    pstree [选项] ：可以更加直观的显示进程信息

    • -p ：显示进程的PID
    • -u ：显示进程的所属用户

    常用组合：ps -aux ，ps -ef

    终止进程的方法

    kill [选项] 进程号 ：通过进程号来杀死进程
    killall 进程名 ：通过进程名杀死进程，也支持通配符。
    常用选项：

    • 9 ：强制执行

    例子：

    1. 关闭sshd服务，其他用户无法远程连接。
    2. 杀死其他用户的sshd进程，踢除正在连接的用户。
    进程动态监控的方法

    top [选项]

    常用选项：

    • -d 秒数 ：指定top指令每个几秒更新一次，默认3s
    • -i ：使top不显示任何闲置或僵死进程
    • -p ：通过指定监控进程ID来仅仅监控某个进程的状态

    交互指令：

    • P ：以CPU使用率排序，默认为改项
    • M ：以内存的使用率排序
    • N ：以PID排序
    • q ：退出top
    • u 用户名 ：过滤出指定user
    • k PID ：杀死某进程

    服务管理

    介绍

    服务本质就是进程，但是运行在后台，同时都会监听某个端口。等待其他程序请求。我们又把他们称为守护进程。

    注意：所有的服务都存放在/etc/init.d/目录下，而这些文件都是可执行的脚本文件，其中对应设置了在不同运行级别下的自启动选项。系统启动时，会自动执行该目录下的所有脚本。

    service 指令

    service 服务名 start|stop|restart|reload|status ：实现对服务的一系列操作，注意这些操作都是临时性的。

    chkconfig --list ：查看所有自启动服务

    chkconfig --level 5 服务名 on/off ：实现不同级别下的服务自启情况

    systemctl 指令

    可以看作是service指令的升级，提供可高效的启动速度。同时能独立实现服务开机自启

    systemctl start|stop|restart|reload|status 服务名

    systemctl enable|disable|is-active 服务名 ：实现服务开机自启或禁用

    systemctl list-unit-files –type=service ： 列出所有服务

    systemctl set-default runlevel（等级1/2/3..）.target ：设置默认运行等级

    添加开机启动项

    假如我们装了一个 zookeeper，我们每次开机到要求其自动启动该怎么办？

    1. 新建一个脚本 zookeeper
    2. 为新建的脚本 zookeeper 添加可执行权限，命令是:chmod +x zookeeper
    3. 把 zookeeper 这个脚本添加到开机启动项里面，命令是：chkconfig --add zookeeper
    4. 如果想看看是否添加成功，命令是：chkconfig --list

    磁盘分区和挂载

    分区方式
    1. 系统只能安装在主分区
    2. 扩展分区要占一个主分区

    加硬盘完整步骤：

    1. 添加一块硬盘 ，重启后查看是否硬盘有成功添加进来。使用lsblk -f 指令来查看磁盘及其分区、挂载情况。

    2. 硬盘做分区：fdisk /dev/sdb

    3. 格式化：往磁盘分区中载入文件系统信息，包括索引节点、初始目录项等，实现初始化。mkfs -t ext4 /dev/sdb1

    4. 挂载：将该分区挂载到某个目录上。实现文件与分区间的关联。mount /dev/sdb1 /home/newdisk 。注意此时仅为临时挂载，一旦重启就又不挂载了。umount 设备名称/挂载目录 ：实现卸载，注意卸载时不要在卸载目录上。

    5. 设置永久挂载：通过直接修改/etc/fstab 配置文件实现永久挂载。修改完后再使用mount -a 指令即可。

    磁盘查看常用指令

    1. df -h ：查看磁盘使用情况，含磁盘大小，使用量，剩余量等信息
    2. du -h 目录 ：查询指定目录的磁盘占用情况
      • -s 指定目录占用大小汇总
      • -a 含文件
      • -c 列出明细同时显示汇总值
      • —max-depth：统计的目录深度

    网络通信命令

    • 查看当前系统的网卡信息：ifconfig/id addr

    • 查看与某台机器的连接情况：ping（网络层）

    • 查看当前系统的端口使用：netstat -anp

      选项：

      • -an ：按一定顺序排列输出
      • -p ：显示那个进程调用
      • -t ：进显示tcp调用的

    包在网络中传输过程，注意IP地址保留封装在包中，MAC地址则会转化为下一跳MAC地址

    • 网络层，查看当前系统的路由情况：route -n

      • route add -host ip1 gw ip2 ：添加一条ip映射（网关）

    • 数据链路层，查看mac映射表：arp -a

    • net-tools 和 iproute2 ： net-tools起源于 BSD 的 TCP/IP 工具箱，后来成为老版本 LinuxLinux 中配置网络功能的工具。但自 2001 年起，Linux 社区已经对其停止维护。同时，一些 Linux 发行版比如 Arch Linux 和 CentOS/RHEL 7 则已经完全抛弃了 net-tools，只支持iproute2。linux ip 命令类似于 ifconfig，但功能更强大，旨在替代它。更多详情请阅读如何在 Linux 中使用 IP 命令和示例

    • shutdown： shutdown -h now： 指定现在立即关机；shutdown +5 "System will shutdown after 5 minutes"：指定 5 分钟后关机，同时送出警告信息给登入用户。

    • reboot： reboot： 重开机。reboot -w： 做个重开机的模拟（只有纪录并不会真的重开机）。
