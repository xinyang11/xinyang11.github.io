Docker 基础学习

一、概念

Docker 容器不使用硬件虚拟化，它的守护进程是宿主机上的一个进程，换句话说，应用直接运行在宿主机内核上。因为容器中运行的程序和计算机的操作系统之间没有额外的中间层，没有资源被冗余软件的运行或虚拟硬件的模拟而浪费掉。

1. 镜像

简单类比为电脑的系统盘，只读。包含有操作环境以及其他应用。

2. 容器

简单类比为电脑的硬件环境。它是真正跑项目，提供服务的东西。基于镜像启动，每个容器之间相互隔离。注意的是，容器在启动的时候基于镜像创建一层可写层作为最上层。

3. 仓库

Docker 的仓库用于存放镜像。这一点，和 Git 非常类似。我们可以从中心仓库下载镜像，也可以从自建仓库下载。同时，我们可以把制作好的镜像 commit 到本地，然后 push 到远程仓库。仓库分为公开仓库和私有仓库，最大的公开仓库是官方仓库 Dock Hub，国内的公开仓库也有很多选择，例如阿里云等。

二、实战

1. 下载镜像

docker pull [image] : image 镜像在https://hub.docker.com/官网仓库上获取。

2. 创建实例并启动

以运行一个redis为例：

docker run ： 创建一个新的容器并运行一个命令

执行过程：通过镜像生成容器

1、查询本地是否存在所需镜像。

2、不存在则前往仓库（registry）下载镜像(pull)

3、通过镜像生成容器

options：

  • -p 端口映射：表示linux下的端口与容器内端口完成映射

  • —name 设置容器名

文件挂载时要注意系统可能回家文件当做是目录，所以在挂载前尽可能先用touch命令创建好文件

  • -v 目录挂载，实现linux下的文件与容器内文件的同步，方便更新查看。

  • -d 后台运行容器，并返回容器ID

基础指令

docker ps : 查看正在运行的容器

docker ps -a ：查看所有容器

docker images ：查看所有镜像

docker rm 容器id/容器名 ：删除容器

docker rmi imageId ：删除镜像

docker restart 容器id/容器名：重启容器

docker logs 容器id/容器名：查看容器日志

docker update 容器id/容器名 --restart=always ：设置自动重启

docker inspect 容器id ：查看容器元数据

docker cp 容器id ：容器内文件拷到主机上

docker commit 容器id [tag] ：提交容器成为一个镜像

  • -m：填写描述信息
  • -a ：作者名

docker build . ：通过dockerFile 生成镜像，最后的 .代表本次执行的上下文路径

  • -f path：指定构建的 dockerFile 文件地址
  • -t 镜像名：设置镜像名

docker login ：远程仓库登录

docker tag ：镜像重命名

docker push imageId ：发布镜像到仓库

docker history imageId ：查看镜像构建过程

  1. 使用实例

    docker exec -it [容器] /bin/bash ：启动容器,转至容器内控制台

    docker attach 容器id ：进入容器正在执行的终端，不会启动新的进程。

    -it ：交互模式下启动

  2. 创建网络

    实现ip->host间的映射

    docker network create ：网络创建

    docker network connect mynet mynginx ：网路建立连接

3. Dockerfile

相信我们现在都已经学会如何使用镜像了。但这些镜像又是怎么来的呢，我们从仓库里拉取的到底是什么？

我们看到拉取文件中就有一个Dockerfile文件。这个文件是干什么的呢？

我们看到这个文件中有多个指令，FROM RUN 。除此之外当然还有其他指令。这些指令有什么用？实质上这与镜像的创建有关。一个镜像包含有操作环境和应用。docker将镜像内部分为多个层级。最下层为基础层，提供基本的操作环境，其上的每一个指令都会构建出新的一层。这多个docker层共同实现了这一个镜像。

Dockerfile指令

  • FROM ：from 后面跟着的是镜像，它将作为我们要创建镜像的基础镜像。同样的若本机没有该镜像将会从仓库远程下载。
  • MAINTAINER ：指明镜像维护者及其联系方式（一般是邮箱地址）
  • RUN ：执行后面跟着的命令，RUN ["可执行文件","参数1","参数2"] 等价于 RUN 可执行文件 参数1 参数2
  • COPY src des ：从上下文路径中复制文件到容器指定路径中（目标路径若不存在会自动创建）
  • ADD src des ：功能与COPY类似。区别：源文件若为tar压缩文件，会自动复制并解压到目标路径中。

CMD 和 ENTRYPOINT 指令若有多条，仅最后一条有效！

  • CMD ：类似RUN，但二者运行时间点不同，RUN执行在docker build时，CMD执行在docker run时。 因此一般用于为启动容器指定默认运行的程序。
  • ENTRYPOINT ：类似CMD，但不会被docker run 额外的指定命令所覆盖，而是追加。
  • ENV key value ：设置环境变量。后续指令可使用该变量。作用域包含整个容器。
  • ARG ：构建参数。作用域仅Dockerfile文件内。
  • VOLUME ["path1","path2"...] ：定义匿名数据卷。
  • EXPOSE ：仅申明端口。注意非端口映射，仅为镜像使用者提供一个查看守护端口的途径。在run中若加入-P 设置随机端口映射时，会指定连接到容器内该端口。
  • WORKDIR ：指定工作目录，注意必须提前创建好。它将在镜像的每一个层级中都存在。
  • ONBUILD 其他指令 ：用于延迟构建命令的执行。当其他镜像设置本镜像为基础镜像时才会触发ONBUILD 后续的指令。
